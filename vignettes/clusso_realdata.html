Running clusso with Areal Data

Introduction

Messages from clusso have been suppressed for this tutorial, but there are many messages generated that we hope the user will find helpful. Feedback is welcome here.

Load clusso package:

library("clusso")

Using clusso with real data (Breast Cancer in Japan)

Prepare Data

We load four datasets that comes with the clusso package:

#load data
data("jbc")
data("utmJapan")
data("japan.poly2")
data("japan.prefect2")

To explore the 4 data sets, we apply the head() function to each of the data sets.

  1. utmJapan: data set containing a unique identifier for each centroid (id), x-coordinate in UTM (utmx), and y-coordinate in UTM (utmy)
  2. japan.poly2: polygon tesselations
  3. japan.prefect2: boundaries for the prefects (geographical boundaries) of the region.
  4. jbc: data set containting a unique identifier for each centroid (id), period of observation (categorical variable with 5 levels, period), death count (death), expected death count (expdeath). These data are simulated (with some noise) based on observed data.

Set global conditions

When using clusso, there are certain global parameters that need to be set.

We take the easting and northing coordinates from the utmJapan dataset, set the easting coordinate, utmx, to x and the northing coordinate, utmy, to y. Each is divided by 1000 to change the scale from meters to kilometers.

x <- utmJapan$utmx/1000
y <- utmJapan$utmy/1000

Below, we set the maximum radius for a cluster to 20 km (rMax=20) based on scientific knowledge of the area and our definition of a meaningful cluster, set the number of unique time periods to 5 (Time=5), and set a floor for overdispersion to avoid estimating underdispersion (overdispfloor=TRUE).

rMax <- 20 
Time <- 5
overdispfloor <- TRUE

If you do not have expected counts readily available, they can be calculated using the auxiliary function calcexpected(), from the clusso package.

To perform the LASSO-based cluster detection, we use the function clusso(). In the first argument, we set the name of the dataframe, df=jbc. In the second argument we set the expected counts equal to the name of expected counts in the jbc dataframe (expected = expdeath). In the third argument, we set the observed counts equal to the name of the observed counts in the jbc dataframe (observed = death). In the fourth argument, we set the timeperiod equal to the name of the time periods in the jbc dataframe and also convert it to a factor in one step (timeperiod = factor(period)). In this example, we do not want to include other covariates as un-penalized terms in the model so we set covars = FALSE.

The next two arguments are x and y, which are the UTM easting and northing coordinates, scaled to be in kilometers. The next argument is the maximum radius (in km) for the potential clusters. We set rMax = rMax, which we defined above as 20km. The argument utm specifies that our coordinates are indeed in UTM coordinates (utm = TRUE). For lat/lon coordinates, we would set utm = FALSE. The is analysis arugment defines the type of analysis to be performed. Options include space, spacetime, or both. Here, we set analysis equal to “both”, which will perform both the spatial and spatio-temporal analyses. The argument model is the type of model to be used. Currently (2019-11-25), either the Poisson or binomial models can be specified. In this example, we specify the Poisson model. The final required argument is maxclust. This is the maximum number of actual clusters expected in the given area. This is based on scientific knowledge of the area. For the Japanese breast cancer data, we set maxclust=11. If the number of identified clusters exceeds maxclust, a warning will encourage you to increase your set maxclust.

The output of clusso() is assigned to the object resreal. resreal is a large list of lists. The recommended way to explore the results is to explore the names(resreal) and select each sub-list of interest.

With the argument clusteridentify=TRUE, centroid indices (by time period) are extracted. Since these lists are very long, output is left to the user to explore:

#identify cluster IDs
clussopretty(resreal, analysis="both", model="poisson",clusteridentify=TRUE)
#identify cluster IDs where RR is greater or equal to 1.05
clussopretty(resreal, analysis="both", model="poisson", clusteridentify=TRUE, clusterRR = 1.05)
#identify cluster IDS that differ from the background rate
clussopretty(resreal, analysis="both", model="poisson", clusteridentify=TRUE, clusterRR = "background")

We can also plot the output using clussoplot() to visualize the identified clusters along the coefficient path:

clussoplot(resreal, analysis="both",model="poisson", Time=5, cv=FALSE)

If we have polygons and boundary files, we can use the easyplot() function. TODO: Current implementation only works for JBC dataset. Functions will be extended to be more generalizable in the future.
